# Launch Readiness Checklist

## Problem and Scope
- problem clearly defined
- target user identified
- success criteria defined
- non-goals documented

## Product / UX
- key user flows mapped
- fallback states defined
- escalation paths defined
- edge cases reviewed

## Technical / Operational
- instrumentation in place
- failure logging available
- dependencies documented
- rollback plan identified

## Metrics
- primary launch metrics defined
- secondary health metrics defined
- monitoring owner assigned
- reporting cadence decided

## Go-to-Market / Enablement
- stakeholder alignment complete
- support teams briefed
- internal documentation prepared
- rollout sequence defined
