# Prioritization Framework

## How I Think About Prioritization

I prioritize work using a simple structure:

### 1. User Pain
How painful is the problem for the user?

### 2. Business Impact
Will solving this improve adoption, retention, efficiency, or revenue?

### 3. Frequency
How often does this problem occur?

### 4. Confidence
How confident are we in the diagnosis and proposed solution?

### 5. Effort / Complexity
How difficult is it to deliver well?

## Example Priority Logic
High priority work often has:
- high user pain
- high business impact
- high frequency
- medium/high confidence
- low/medium implementation effort
